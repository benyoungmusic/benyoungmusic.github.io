About Me

I love solving puzzles. This drew me to music composition, where I would incorporate the creative vision of a filmmaker or game dev, and the implicit musical associations of an audience to enhance emotions, emphasize a story, or amplify an actor's performance. My love of puzzles then led me to game development, where I coded adaptive music logic into game engines, QA tested, mediated between departments, and managed budgets and schedules. From game development, I moved to perhaps the ultimate in puzzle solving, software development. As a software engineer there are far more interesting puzzles to solve and I get to juggle considerations like time and space complexity, UI, UX, and a variety of frameworks, in several different languages.

Skills

Projects

Choosy

A decision making app

Choosy is a mobile-friendly app that helps individuals and groups quickly make decisions, built using the MERN stack. I constructed a custom dual-timer-based algorithm leveraging Javascript’s setInterval function that dictates Choosy's UX flow and controls child components. My 3 other team members and I designed, wire-framed, documented and built the core functionality of the app in a week.

Digital Cities

A futuristic city builder

Digital Cities is a simple, visual game I built in vanilla JS and canvas that gives players a set of futuristic roads and buildings to create a city of their choosing. I integrated a universal "bloom" effect compatible across browsers using an extra layer of canvas, duplicate triggers and the CSS "blur" effect. I also animated the roads using vectors created from the user's start and end points

Trellis

A trello clone

Trellis is a clone of the project management site/app “Trello” I built using Ruby on Rails, PostgreSQL, and React.js/Redux. I implemented drag-and-drop functionality for cards using HTML drag and drop API that updates the cards' position in the database through several components. I also disguised the card details page to allow a card-specific link to be saved without interrupting page flow.
